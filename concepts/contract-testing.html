Contract Testing

Definition

Contract testing is a testing methodology that ensures API responses match SDK type definitions and SDK requests comply with API validation rules. It validates the "contract" between services without requiring full integration tests.

Purpose

Contract testing serves as the critical mechanism for type consistency between the CE Orchestrator API and its SDK. Through automated testing, it verifies:

  1. API Response Consistency - API responses match the type definitions expected by the SDK
  2. Request Validation - SDK requests comply with API validation rules
  3. Error Response Standardization - Error responses follow consistent formats across all endpoints

Advanced Testing

Boundary Conditions

  • String length boundaries (1 char, 200 chars, Unicode)
  • Numeric boundaries (limit=0, limit=999999)
  • Array boundaries (empty array, large arrays)
  • Optional field handling
  • Null/Undefined handling
  • Deep nested objects
  • ID format boundaries
  • Timestamp format validation

Concurrent Requests

  • Parallel read operations
  • Parallel write operations
  • Mixed read/write operations
  • Parallel updates to same resource
  • Rate limiting and throttling
  • Race condition handling
  • Authentication token reuse

Performance Testing

  • GET request performance (simple/complex)
  • POST request performance (create operations)
  • PUT request performance (update operations)
  • DELETE request performance (delete operations)
  • Concurrent request performance
  • Large response handling
  • Authentication performance
  • Memory efficiency checks

CI/CD Integration

Location: .github/workflows/contract-tests.yml

Contract tests run automatically:

  • On push/PR to main/develop branches
  • Manual trigger support
  • Complete test result upload and reporting

Benefits

  1. Type Safety - Ensures API and SDK types remain consistent
  2. Early Detection - Catches breaking changes before deployment
  3. Documentation - Serves as living documentation of API contracts
  4. Confidence - Provides confidence in API stability
