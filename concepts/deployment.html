Deployment

Definition

Deployment refers to the process of making the CE Orchestrator application available for use, whether in development, staging, or production environments.

Prerequisites

Required Software

Software Version Purpose
Node.js v22+ Runtime environment
pnpm v9+ Package manager
Docker v24+ Container management
Git - Version control

System Requirements

Resource Minimum Recommended
CPU 2 cores 4 cores
Memory 4GB 8GB
Storage 20GB 50GB
Network Ports 9900, 8000 Same

Deployment Modes

1. Development Deployment

Docker Compose (Development)

# Start with hot reload
docker-compose -f docker-compose.dev.yml up --build

# Run in detached mode
docker-compose -f docker-compose.dev.yml up -d --build

# View logs
docker-compose -f docker-compose.dev.yml logs -f

# Stop
docker-compose -f docker-compose.dev.yml down

Local Development

# Terminal 1: Start API server
pnpm dev

# Terminal 2: Run tests
pnpm test:unit:watch

# Terminal 3: Run contract tests
pnpm test:contract:watch

2. Production Deployment

Building for Production

# Install dependencies
pnpm install --prod

# Generate API routes
pnpm build:api

# Build TypeScript
pnpm build

# Or combine all
pnpm build

Running in Production

# Using Node.js
NODE_ENV=production pnpm start

# Using PM2 (recommended)
pm2 start dist/index.js --name ce-orchestrator

# Using Docker
docker-compose up -d

3. Docker Deployment

Production Docker Compose

version: "3.9"

services:
  orchestrator:
    build:
      context: .
      dockerfile: Dockerfile
    ports:
      - "${PORT:-9900}:9900"
    env_file:
      - .env.local
    environment:
      - NODE_ENV=production
    volumes:
      - /var/run/docker.sock:/var/run/docker.sock
      - ./data:/app/data
    networks:
      - kai-net
    restart: unless-stopped
    deploy:
      resources:
        limits:
          cpus: '2'
          memory: 4G
        reservations:
          cpus: '1'
          memory: 2G

networks:
  kai-net:
    driver: bridge

Docker Run Command

docker run -d \
  --name ce-orchestrator \
  --restart unless-stopped \
  -p 9900:9900 \
  -v /var/run/docker.sock:/var/run/docker.sock \
  -v $(pwd)/data:/app/data \
  -e NODE_ENV=production \
  -e JWT_SECRET=your-secret-key \
  --network kai-net \
  ce-orchestrator:latest

Environment Configuration

Required Variables

Variable Description Default
PORT Server port 9900
NODE_ENV Environment mode production
JWT_SECRET JWT signing secret Required

Optional Variables

Variable Description Default
FRONTEND_URL CORS allowed origin *
DOCKER_NETWORK Docker network name kai-net
KAI_BASE_ROOT Data storage path /tmp/KaiBase
CONTEXT_ENABLED Enable context manager false
CONTEXT_MANAGER_URL Context manager URL http://localhost:8080
ROUND_TABLE_URL Round Table server URL -
LOG_LEVEL Logging level info

Health Checks

HTTP Health Endpoint

curl http://localhost:9900/api/health

Response:

{
  "status": "healthy",
  "timestamp": "2026-04-17T12:00:00.000Z",
  "uptime": 3600
}

Docker Health Check

docker inspect --format='{{.State.Health.Status}}' ce-orchestrator

Security Checklist

  • Change JWT_SECRET to a strong random value
  • Set NODE_ENV=production
  • Disable DEBUG_MODE
  • Configure FRONTEND_URL for CORS
  • Use HTTPS in production
  • Set up proper firewall rules
  • Regular security updates
  • Monitor logs for suspicious activity

Scaling

Horizontal Scaling

Deploy multiple instances behind a load balancer for high availability:

services:
  orchestrator:
    deploy:
      replicas: 3
    # ... other config

  nginx:
    image: nginx:alpine
    ports:
      - "80:80"
    volumes:
      - ./nginx.conf:/etc/nginx/nginx.conf
    depends_on:
      - orchestrator

Backup and Recovery

Data Backup

# Backup data directory
tar -czf backup-$(date +%Y%m%d).tar.gz ./data

# Backup environment configuration
cp .env.local .env.local.backup

Restore

# Extract backup
tar -xzf backup-20260417.tar.gz

# Restore environment
cp .env.local.backup .env.local
