API Endpoints

Description

API endpoints are the primary interface for interacting with the CE Orchestrator. All endpoints are documented with OpenAPI 3.0 specifications and include authentication requirements, request/response schemas, and error handling documentation.

Documentation

Swagger UI (Interactive)

  • Development: http://localhost:9900/api-docs
  • Production: /api-docs endpoint
  • Features: Interactive API testing, request/response examples, "Try it out" functionality

ReDoc (Reference)

  • Development: http://localhost:9900/redoc
  • Production: /redoc endpoint
  • Features: Clean, responsive design, three-panel layout, search functionality

Static HTML Documentation

# Generate standalone HTML documentation
pnpm build:docs
# Output: dist/redoc-static.html

Authentication

All protected endpoints require a valid JWT Bearer token:

Authorization: Bearer <your-jwt-token>

Public endpoints (no authentication required):

  • /auth/login
  • /auth/refresh
  • /auth/verify
  • /health
  • /api/health
  • /context/health

Error Responses

All error responses follow a consistent format:

{
  "error": {
    "code": "ERROR_CODE",
    "message": "Human-readable error message",
    "details": {}
  }
}

Contract Testing

All endpoints are covered by contract tests to ensure API-SDK type consistency. See Contract Testing for details.
